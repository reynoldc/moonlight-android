# 小米平板触摸板兼容模式指南

## 目标

- 启用新增的“小米平板触摸板兼容模式”
- 尽量保留 HyperOS 官方触摸板手势，让双指滚动、双指右键可用

## 1. 安装打好的 APK

- 将 `moonlight-axi-nonRoot-release.apk` 传到小米平板
- 点击安装
- 如果系统提示未知来源，先允许安装

## 2. 打开设置入口

- 应用内路径：

```text
Moonlight
-> 设置
-> 输入设置
-> 小米平板触摸板兼容模式
```

## 3. 开启开关

- 打开 `小米平板触摸板兼容模式`
- 当前实现默认就是开启的，但建议手动确认一次

## 4. 检查系统侧设置

- 到小米平板系统设置中确认：
- 已开启“轻触单击”
- 已连接官方触控键盘或官方触摸板
- 系统指针本身能正常使用

## 5. 重新进入串流

- 建议先退出当前串流会话，再重新进入
- 优先进入 Windows 桌面环境测试，不要先进入游戏

## 6. 建议测试项

- 单指移动：远端鼠标是否正常移动
- 单击：是否是左键
- 双指轻敲：是否变成右键
- 双指滑动：是否变成滚轮
- 连续操作：是否有异常跳动、失焦、错位

## 7. 如果效果不对

- 先关闭这个兼容模式，再重新进入串流对比一次
- 如果关闭后恢复旧行为，说明问题确实在兼容模式链路
- 如果开关前后都没有差异，再检查：
- 系统“轻触单击”是否已开启
- 当前连接的是否为官方触控键盘
- 是否启用了其他会影响鼠标输入的特殊模式

## 8. 兼容模式原理

- 开启后：
- 对小米/红米/Poco 平板，不优先启用 `requestPointerCapture()`
- 尽量保留系统层已实现的触摸板手势输入
- 让应用内现有触摸板逻辑更容易收到双指滚动、双指右键相关事件

- 关闭后：
- 恢复 Moonlight 原本的鼠标捕获逻辑

## 9. 可能出现的现象

- 本地鼠标光标和远端鼠标光标可能同时显示
- 某些桌面场景下滚动手感可能与普通鼠标略有不同
- 这属于该兼容方案可能带来的副作用，和参考改法一致

## 10. 相关代码位置

- 开关定义：`app/src/main/res/xml/preferences.xml`
- 配置读取：`app/src/main/java/com/limelight/preferences/PreferenceConfiguration.java`
- 生效逻辑：`app/src/main/java/com/limelight/binding/input/capture/InputCaptureManager.java`
